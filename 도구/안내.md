# 도구 (Tools)

**간단한 원자적 기능들의 집합.** 곱셈·뺄셈처럼 하나의 일만 하는 기본 함수/유틸리티.
[[스킬]]은 이 도구들을 조합해서 만든다 — 도구가 부품, 스킬이 조립품이다.

## 하위 구성 (모든 자원 폴더 공통)
- **기본** — 워크스페이스 구동에 *반드시 필요한* 것. → **Git 추적(깃허브에 올라감).**
- **추가** — 일반 확장. → 로컬 전용(Git 제외).
- **고급** — 고급·특수 용도. → 로컬 전용(Git 제외).
- **대외비** — 민감·비공개. → 로컬 전용(Git 제외).

## 도구 만들기 규약 (필수)

도구는 **범용·재사용** 기능만 둔다 — "여러 스킬·작업이 또 쓸 만한가?"가 Yes인 코드. 특정 스킬 전용 로직은 그 스킬의 `scripts/`에.

### 폴더 구조
```
도구/<등급>/<tool-name>/
├── 도구.md     (필수 — 매니페스트)
└── <코드>       (실제 기능, 단일 책임)
```

### 매니페스트 `도구.md` (frontmatter)
```yaml
---
name: <tool-name>
description: <무엇을 + 언제 쓰는지 + 사용자 표현 예시 + 핵심 용어>   # 발견기가 잘 찾게
entry: <진입 파일[:함수]>      # 예: multiply.py:multiply
runtime: <python|node|sh ...>
---
```

### 발견 (태그 없음)
- 도구는 **frontmatter `description`으로 발견**된다. 별도 등록·태그가 필요 없다 — 발견기가 매번 스캔한다.
- 그래서 `description`을 위 형식대로 또렷이 써야 잘 찾힌다. 이게 발견 품질을 좌우한다.

### 다른 문서에서 이 도구 참조 (law.md §8)
특정 문서를 박지 말고 **자연어로 적고 발견기로 찾는다**: `관련 도구: 두 수를 곱하기 — 발견기로 찾아 사용`. (반드시 확정해야 할 때만 경로 직접 참조.)

> 도구를 만들고 고치는 자세한 절차는 [[스킬]] `스킬/기본/tool-creator`. 스킬에서 범용 기능이 필요할 때 도구로 빼는 판단은 `skill-creator`의 "도구 우선" 절도 참조.
