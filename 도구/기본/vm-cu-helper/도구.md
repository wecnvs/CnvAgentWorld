# vm-cu-helper (VM 안에서 도는 컴퓨터유즈 헬퍼)

여러 에이전트가 같은 호스트 화면을 두고 충돌하는 문제를, **각 VM의 자체 화면에서 CU를 실행**해 푼다. VM마다 이 헬퍼를 그 VM의 대화형(콘솔) 세션에서 돌리면, 호스트 에이전트가 HTTP로 **그 VM의 화면만** 캡처·조작한다 → 서로 다른 VM의 두 에이전트가 **동시 병렬 CU**해도 충돌 0. 호출하는 쪽(호스트 클라이언트)은 도구 묶음 `cu-win`의 `vm_cu.py`이고, 컴퓨터유즈 절차·안전 규칙은 스킬 `computer-use-win`을 따른다.

## 구성
| 파일 | 한 일 |
|---|---|
| `cu_helper.ps1` | VM 안 데몬(PowerShell/.NET 자립형, **python 불필요**). ★대화형 데스크톱 세션에서 실행돼야 캡처·입력이 동작(서비스/비대화형이면 검은 화면·입력 무시). 기동: `powershell -ExecutionPolicy Bypass -File cu_helper.ps1 [-Port 8599]` |
| `vms.template.json` | VM 레지스트리 **템플릿(placeholder만)**. name=CU락 target, host/port. ★실제 값은 **대외비** — 아래 보안 참조. |
| `transcode_proxy.py` | **호스트에서 실행(Python+Pillow)** — VM 원격뷰 실시간화. VM cu_helper에 JPEG를 넣으면 **VM Windows Defender AMSI가 'ScriptContainedMaliciousContent'로 차단**(화면캡처→JPEG 패턴)하므로, VM엔 PNG 헬퍼를 두고 **호스트가 VM PNG를 빠른 로컬링크로 받아 여기서 축소+JPEG 변환**(2.5MB PNG→~90KB JPEG, 27배↓) 후 작은 JPEG만 Tailscale로 보낸다. `/screenshot`만 변환, 나머지는 패스스루. 구조: Mac→Tailscale→**netsh portproxy(8601/8602→127.0.0.1:18601/18602)**→프록시(localhost 바인딩)→VM:8599. 영속화=스케줄작업 `CAW-Transcode-Proxy`(AtStartup). 기동: `python transcode_proxy.py --map 18601=<VM-A NAT IP> 18602=<VM-B NAT IP>`. ※VM NAT IP가 재부팅으로 바뀌면 이 --map(스케줄작업 인자)을 갱신해야 함. 프록시 활성 시 portproxy는 127.0.0.1을 가리키므로 recover_vm.ps1의 portproxy 갱신과 충돌 — 프록시 셋업이 portproxy를 소유. 잔여지연(~1.2s/프레임)은 VM 풀해상도 PNG 인코딩 비용(네트워크 아님) → 더 빠르게 하려면 VM 해상도 하향(/resolution). |
| `recover_vm.ps1` | **호스트에서 실행** — Hyper-V 게스트 VM의 cu_helper를 안전하게 (재)설치·복구. 헬퍼 ParseFile 선검증 → PSDirect 확보(안 되면 **IC 토글**, 그래도 안 되면 graceful 재부팅) → 정상헬퍼 설치+CUHelper 작업 기동 → (선택)포트프록시 갱신. **강제재부팅 남발 금지**가 설계에 박혀 있음(2026-06-30 BB-Win11-2 복구 정본화). cred는 `자산/대외비/vm-targets/vm_credentials.json`. ※UTF-8 **BOM 필수**(PS 5.1이 한글 주석을 ANSI로 읽어 깨지는 함정 방지). |

## HTTP API (`http://<VM_HOST>:<PORT>/`, 기본 8599)
| 경로 | 설명 |
|---|---|
| `GET /status` | `{ok,hostname,user,screen{w,h,x,y},cursor{x,y},session}` |
| `GET /screenshot[?cursor=0]` | image/png (기본: 빨강 커서마커 오버레이) |
| `GET /cursor` | `{x,y}` |
| `POST /move {x,y}` | 커서 이동(클릭 없음) |
| `POST /click {x,y,button,double}` | 클릭(button: left\|right\|middle) |
| `POST /scroll {x,y,amount}` | 휠(+위 / −아래, notch) |
| `POST /type {text}` | 클립보드 경유 붙여넣기(한글/특수문자 안전) |
| `POST /key {keys}` | SendKeys 문법 키조합(예 `^c`, `{ENTER}`, `%{F4}`) |
| `POST /resolution {w,h}` | 디스플레이 해상도 변경(ChangeDisplaySettings) |
| `POST /run {cmd,cwd}` | 이 세션에서 프로그램 실행(GUI앱 포함) → `{ok,pid}`. **대시보드 앱 탭의 '실행'**(target=cu-helper)이 호출. Win32_Process.Create로 실제 pid 반환 |
| `POST /ps {pid}` | 그 pid 생존확인 → `{ok,alive}` (앱 탭 실행상태 폴링) |
| `POST /stop {pid}` | 그 pid(+자식) 종료(`taskkill /T /F`) → `{ok}` (앱 탭 '중지') |

> `/run·/ps·/stop`은 **앱 레지스트리**(`core/apps.py`, target 채널 `cu-helper`)가 호출한다 — 원격 윈도우/VM의 앱을 대시보드 앱 탭에서 실행·중지·추적하기 위함. CU 조작(클릭/타이핑)과 별개로 '프로그램 띄우기'를 담당한다.

> 호스트 에이전트는 이 API를 직접 치지 말고 `cu-win`의 `vm_cu.py <target> <cmd>`로 호출한다(+ per-VM `cu_lock.py --target <vm>`). `<target>`은 레지스트리의 name(또는 `host:port`).

## 레지스트리 해석 순서 (vm_cu.py가 자동 탐색 — 절대경로 하드코딩 없음)
1. `도구/대외비/vm-cu-helper/vms.json` ← **실제 host/port(대외비, 깃 미추적)**
2. `도구/기본/vm-cu-helper/vms.json` ← 운영자가 채워둔 실제 레지스트리(있으면)
3. `도구/기본/vm-cu-helper/vms.template.json` ← 공개 템플릿(placeholder, 실제 연결 X)

## 보안 (law.md §7 보안 불변식 재매핑)
- **실제 VM host/port·VM명·자격증명은 대외비(타깃주소·연결정보)다.** 공개 경로(`도구/기본/vm-cu-helper/`)에는 **템플릿/스키마(placeholder)만** 두고, 실제 연결정보는 `도구/대외비/vm-cu-helper/vms.json` 등 **대외비 등급**에 두어 발견기로 참조한다. 이 폴더의 어떤 파일·로그·캡처에도 실제 IP/호스트/비밀번호를 평문으로 남기지 않는다.
- **헬퍼 (재)배포·VM 복구**는 이제 워크스페이스 자체 도구 `recover_vm.ps1`로 한다(brainbase `deploy_cu_helper.ps1` 의존 끊음). 자격증명은 `자산/대외비/vm-targets/vm_credentials.json`(VmCredential 형식 `{accounts:[{user|username|name, password|pass|pwd}]}`, 깃 미추적). 관리자/맥 쪽은 cred+헬퍼+`recover_vm.ps1`을 호스트 ASCII 임시경로로 scp→실행→**cred 임시본 즉시 삭제**한다. VM 기동·자동로그온·자격 배치 같은 되돌리기 어려운 작업은 여전히 사전 고지·범위 승인 후 수행한다.
- 헬퍼 접속 실패(미기동)면 CU를 시작하지 말고 보고한다.

> 절차·실패 분기·실전 교훈은 스킬 `computer-use-win`(SKILL.md/cases.jsonl)을 따른다. 컴퓨터유즈 공통 원칙은 `computer-use-charter` 헌장을 먼저 적용한다(발견기로 찾아 적용).
