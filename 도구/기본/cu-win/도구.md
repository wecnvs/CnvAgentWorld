# cu-win (도구 묶음)

Windows 컴퓨터유즈를 위한 **원자 기능 17종**. 스킬 `computer-use-win`이 이 도구들을 조합해 호출한다(스킬은 방법, 이 폴더는 부품). 실제 조작 절차·안전 규칙·실전 케이스는 스킬 쪽(`SKILL.md`/`cases.jsonl`)을 따르고, 여기서는 도구의 호출 인터페이스만 정의한다.

## 실행 래퍼 (항상 이걸로 호출)
```bat
도구\기본\cu-win\run_tool.bat <script.py> [인수...]
:: PowerShell: 도구\기본\cu-win\run_tool.ps1 <script.py> [인수...]
```
- `run_tool.bat`/`run_tool.ps1`은 자기 폴더 기준으로 python·도구 경로를 잡아 실행한다 → 폴더 위치가 바뀌어도 동작.
- Windows 전용 의존: pyautogui·mss·pywin32·pyperclip 등(머신 환경 의존, 비밀값 아님). 환경 점검·설치는 원본 `scripts/`(check_env.py·install_deps.bat) 절차를 발견기로 찾아 1회.

## 안전·보안 (스킬에서 강제)
- **동시 화면충돌 방지**: 물리 화면이 1개라 GUI 조작 전 `cu_lock.py acquire`(VM이면 `--target <vm>` — per-VM이라 다른 VM은 병렬), 긴 작업 중 `heartbeat`, 끝나면(실패·중단 포함) 반드시 `release`.
- **안전 클릭**: 추측 클릭 금지 — `move → screenshot(빨강 포인터 마커가 목표 위인지 눈확인) → click`. 작은 버튼·웹 요소는 `find_element.py`/`web_locate.py`로 정확 좌표 조회.
- **VM 원격(`vm_cu.py`)**: `<vm>` target은 레지스트리(`vm-cu-helper/vms.json`)의 name으로 해석한다. **실제 VM host/port·자격증명은 대외비** — 공개 경로엔 템플릿만 두고 실제 값은 `도구/대외비/vm-cu-helper/vms.json`에 두어 vm_cu.py가 대외비→기본→템플릿 순으로 자동 탐색한다(절대경로 하드코딩 없음).
- **비밀값**(비밀번호·OTP·토큰·원격 연결정보·VM host/port)은 인수/로그/캡처 어디에도 평문으로 남기지 않는다(law.md §7 보안 불변식). 민감 화면 캡처는 공개 경로에 두지 않는다.
- 실제 원격제어 인프라(헬퍼 배포·VM 기동·터널/방화벽·자격증명 배치)는 이 도구 참조만으로 진행하지 않고 **별도 결재** 후 수행한다.

> 호출 절차·실패 분기·실전 교훈은 스킬 `computer-use-win`을 따른다. 컴퓨터유즈 공통 원칙은 `computer-use-charter` 헌장을 먼저 적용한다(발견기로 찾아 적용).
