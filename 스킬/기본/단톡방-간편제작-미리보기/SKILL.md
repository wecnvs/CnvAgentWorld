# 단톡방-간편제작-미리보기

이 스킬은 챗봇(에이전트)이 단톡방이나 대화 공간에서 사용자의 간단한 제작 요청을 처리하고, 결과를 말풍선 안에서 렌더링된 시각적 '미리보기'로 제공하기 위한 지침입니다.

## ⚠ 복잡도 안내
다음 중 하나라도 해당되면 **상위 모델 사용 권장**이나 전용 스킬을 먼저 탐색하세요:
- [ ] 단순한 한 페이지/간단한 스크립트가 아니라 수십 페이지에 달하는 대규모 보고서나 전체 웹사이트 프로젝트를 빌드해야 하는 경우
- [ ] 정합성이나 오류 검증을 위한 복잡한 테스트가 수반되는 작업
> "이 작업은 단순한 간편 제작 범위를 넘어서므로, 전용 스킬(예: 보고서 작성기, 프로젝트 빌더 등)이나 상위 모델을 사용하는 것이 좋습니다. 그래도 간단한 형태로 진행할까요?"

## 🎯 핵심 교훈 (필독)

**산출물 파일을 생성한 뒤 대화방에 결과를 보고할 때:**
- ❌ **하지 말 것**: 결과물 내용을 단순히 마크다운 코드블록(```html, ```markdown 등)이나 텍스트 형태로만 출력하기
- ✅ **반드시 할 것**: 사용자가 눈으로 직접 확인할 수 있도록 **말풍선 내에서 실제 렌더링된 시각적 미리보기 형태**로 제시하기

**특히, 마크다운(md) 파일을 생성하고 말풍선에 미리보기로 제공해야 할 때:**
- ❌ **하지 말 것**: 파일 경로만 단순 텍스트로 적어두거나, 루트 상대경로 문자열만 던지기
- ✅ **반드시 할 것**: **반드시 `[파일명](file:///절대경로)` 형태의 마크다운 링크 문법을 사용**해야 대시보드(말풍선)에서 렌더링 및 미리보기가 정상적으로 동작합니다. 절대경로를 구성할 때는 현재 작업 폴더(`cwd`)를 기준으로 전체 절대경로를 조합하세요.

**예시**:
- ❌ 틀림: "완료했습니다. 다음은 생성된 문서입니다:\n```markdown\n# 제목\n내용\n```" (코드블록에 가둠)
- ❌ 틀림: 절대경로(`file:///...`)나 작업폴더 밖·불완전 경로 — 대시보드가 렌더 못 한다.
- ✅ 맞음: **루트폴더 기준 상대경로**를 한 줄로 적는다(절대경로·`file://` 금지). 예: `결과보고서 만들었어요 — 에이전트/<내토큰>/공간/<방>/작업/<작업코드>/결과보고서.md` → 대시보드가 그 경로를 말풍선 미리보기로 자동 렌더한다.

## 언제 쓰나
단톡방에서 대표와 대화하다가 "간단한 걸 만들어서 보여줘", "이미지/그림/표/카드/문구/배너 만들어줘", "미리보기로 보여줘"처럼 **가벼운 산출물을 만들어 그 자리에서 보여달라**는 요청일 때. (큰 프로젝트·정식 문서는 해당 전용 스킬을 쓴다 — 이건 '대화 중 간편 제작'용.)

## 핵심 원칙
- **간단한 요청엔 간단하게.** 과하게 무겁게 만들지 않는다. 가장 가벼운 형식을 고른다.
- **결과는 반드시 말풍선에서 보이게 한다.** 텍스트는 인라인, 시각물 및 생성된 파일은 명시적인 미리보기 링크 및 렌더링 지원 형식으로.
- **대표가 형식을 지정하면(html·md/마크다운·pdf·이미지·문서 등) 그 형식으로 만들어 반드시 말풍선 미리보기로 보여준다.** 

## 자연어 → 행동 변환 가이드 (에이전트가 수행)

| 사용자가 한 말 | 에이전트의 판단 및 행동 |
|----|----|
| "오늘 회의 요약 좀 텍스트로 바로 적어줘" | 작업 폴더에 파일 생성 없이 즉답으로 말풍선에 요약 내용 작성 |
| "이번 달 실적을 표로 그려서 간단히 이미지로 보여줘" | 이미지 렌더링 스킬을 사용해 .png 파일 생성 후 `[실적.png](file:///절대경로/실적.png)` 형태 등으로 답변 작성 |
| "논의 결과를 마크다운(md) 문서로 정리해서 미리보기 띄워봐" | 작업 폴더에 `.md` 파일 생성 후 반드시 `[논의결과.md](file:///절대경로/논의결과.md)` 형태로 말풍선 작성 |
| "랜딩 페이지 초안을 html로 만들어서 줘" | HTML 파일 생성 후, 코드블록이 아닌 `[초안.html](file:///절대경로/초안.html)` 링크나 렌더 경로로 제공 |

**기본값**:
- 사용자가 별도 형식을 명시하지 않으면, 간단한 목록은 텍스트로, 복잡한 구조나 긴 텍스트는 `.md` 마크다운 파일로 생성하여 `[파일명](file:///절대경로)` 제공.

## 절차
1. **트리아지:** 한두 줄 텍스트(슬로건·문구·요약·목록)면 작업화하지 말고 **말풍선에 바로 적어 답한다**. 그 외 시각물(이미지·HTML·md 파일 등)이면 아래 절차를 따른다.
2. **가벼운 형식 선택:** SVG/PNG 이미지, 작은 HTML 한 장, 마크다운 표 중 가장 적합하고 가벼운 것. 
3. **만들기:** 작업에이전트에게 만들게 하거나 파일을 직접 작성한다. 산출물은 작업폴더에 저장한다.
4. **미리보기 제공:** **⚠️ 주의**: 결과물을 마크다운 코드블록으로 감싸서 제시하지 말 것. 
   - 마크다운(md), PDF, 일반 파일인 경우 반드시 **`[파일명](file:///절대경로)`** 포맷의 마크다운 링크 문법으로 작성한다. 
   - 이미지, 영상 등 기타 자동 렌더링 지원 포맷도 시스템이 렌더할 수 있도록 경로를 적절히 남긴다.
5. **수정 루프:** 사용자가 피드백을 주면 파일을 수정하고 링크를 다시 제시한다.

## 미리보기 지원 형식 (대시보드가 자동 렌더)
- **이미지**: png · jpg · jpeg · gif · webp · bmp · svg → 말풍선 인라인 이미지
- **영상/오디오**: mp4 · webm · mov / mp3 · wav · ogg → 플레이어
- **문서(바로)**: pdf · html → iframe 미리보기
- **텍스트/마크다운**: md · txt · csv · json · yaml · log → 가급적 `[파일명](file:///절대경로)` 링크로 첨부하여 인라인 렌더 및 다운로드 가능케 함.
- **오피스**: pptx · ppt · docx · doc · xlsx · xls · odp/odt/ods → 서버가 자동 **PDF로 변환**해 iframe 미리보기
- **미리보기 불가**: zip · hwp · 7z 등 → 파일 카드(다운로드 링크). 이런 건 가능하면 미리보기 되는 형식(pdf/png/html)으로도 같이 낸다.

## 안티패턴 (반드시 피하기)
- ❌ **마크다운 코드블록으로만 출력하기** — ```html이나 ```markdown 같은 코드블록에 결과를 가두지 말 것. 반드시 실제 렌더링된 시각적 미리보기(이미지/iframe/파일카드)로 제시할 것.
- ❌ **MD 파일의 단순 경로 텍스트 표기** — `결과.md 생성됨: /경로/결과.md` 처럼 텍스트만 적으면 안 됨. 반드시 `[결과.md](file:///경로/결과.md)` 링크 사용.
- ❌ 간단 요청을 무거운 빌드로 부풀리기
- ❌ 파일 경로만 적고 내용 미리보기는 안 포함하기
