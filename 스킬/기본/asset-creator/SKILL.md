# Asset Creator

`자산/`에 **계속 쌓아 관리하는 보유 자원**(ID·API키·자료 등)을 등록한다. 자산은 스킬·지식이 필요할 때 **참조**한다(law.md §8).

## 비밀 취급 (가장 중요)
- API키·자격증명 등 **비밀 '값'은 `자산/대외비/<이름>/`에** 둔다(폴더째 Git 제외).
- `자산.md`에는 **설명만** 적고 **비밀 값은 절대 적지 않는다.**
- 코드는 비밀을 런타임에 상대경로·환경변수로만 읽는다. 절대경로 하드코딩 금지. 예시 설정은 `*.example`로만.

## 만들기 절차
1. **등급 선택**: `자산/{기본|추가|고급|대외비}/<asset-name>/` (비밀이면 대외비).
   > **🔒 대외비 (보안 — law.md §7):** 자격증명·API키뿐 아니라 **개인정보·특정 회사/고객/기관 데이터**도 `대외비/`. 공개 깃엔 `기본/`만 올라가니 민감 자산이 `기본/`에 들어가면 유출이다. 애매하면 대외비.
2. **작성**:
   ```
   자산/<등급>/<asset-name>/
   ├── 자산.md     # frontmatter(name, description) — 비밀 값 금지
   └── <값/데이터>  # 비밀이면 대외비 폴더라 Git 제외
   ```
   `자산.md` frontmatter:
   ```yaml
   ---
   name: <asset-name>
   description: <무슨 자산인가 + 언제 쓰나 + 핵심 용어 — 비밀 값은 쓰지 않는다>
   ---
   ```

## 발견 잘 되게 — `description`이 전부 (태그 없음)
자산은 별도 등록·태그 없이 **frontmatter `description`으로 발견기에 잡힌다.** description을 *무슨 자산인지 + 언제 쓰는지 + 핵심 용어*로 또렷이 쓰되 **비밀 값은 넣지 않는다.** 대외비 자산은 폴더가 Git에서 빠져, 클론에 따라 실제 폴더가 없을 수 있다 → 참조 측은 "없구나" 하고 차선(law.md §6).

## 다른 문서가 이 자산을 쓰는 법 (law.md §8)
`관련 자산: OpenAI API 키 — 발견기로 찾아 사용`처럼 **자연어로** 적고, 사용 시점에 발견기로 찾는다.

## 완료 기준 (DoD)
- [ ] `자산/<등급>/<이름>/자산.md`(name·description) 실제 생성, **비밀 값 미포함**.
- [ ] 비밀 '값'은 `자산/대외비/`에만, 코드는 환경변수·상대경로로 읽음.
- [ ] `description`이 발견 최적화 형식으로 또렷함.
