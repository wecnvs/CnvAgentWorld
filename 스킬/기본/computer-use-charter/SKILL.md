# computer-use-charter (컴퓨터유즈 공통 헌장)

이 스킬은 어떤 엔진·에이전트든 컴퓨터 화면을 직접 조작할 때 가장 먼저 적용하는 **공통 헌장**이다. macOS 컴퓨터유즈, 브라우저 제어, Parallels/Windows VM, Autodesk/Revit 로그인, 대시보드 UI 조작에 공통으로 적용한다. 앱별 세부 스킬이 따로 있으면, 이 헌장을 먼저 적용한 뒤 해당 스킬을 읽는다.

## 언제 쓰나
- 화면을 보고 클릭/타이핑해야 할 때(컴퓨터유즈), 앱을 실행할 때, 브라우저를 조작할 때.
- 로그인·OTP 입력, Revit 실행, 마우스/키보드 자동화, 원격 데스크톱/VM 화면 제어를 할 때.
- "스크린샷으로 증거를 남겨라" 같이 화면 변화를 관찰·검증해야 하는 모든 GUI 작업.

## 원칙
화면 조작은 **추측 실행이 아니라 관찰 기반 작업**이다. 모든 엔진은 성능 수준과 무관하게 `캡처 → 판단 → 행동 → 캡처 → 검증` 루프를 따른다. 경량 엔진(로컬 모델 등)도 이 순서를 생략하지 않는다 — 경량화는 허용하되 원칙 생략은 허용하지 않는다.

## 표준 루프
1. **Lock**: 화면을 조작하기 전, 가능하면 컴퓨터유즈 락(동시 조작 방지)을 먼저 잡는다. 락을 잡을 수 없으면 클릭·타이핑을 시작하지 않는다.
2. **Observe**: 행동 전 스크린샷을 저장하고 OCR/비전/DOM/프로세스 상태로 현재 화면을 판정한다.
3. **Plan**: 목표 UI가 실제로 보이는지, 포커스가 맞는지, 팝업·로그인·로딩·만료·잠금 상태인지 가설을 세운다.
4. **Move**: 클릭이 필요하면 먼저 포인터만 목표 위로 이동한다.
5. **Confirm**: 이동 후 다시 캡처해 포인터가 목표 위에 있는지 확인한다. 확실하지 않으면 클릭하지 않는다.
6. **Act**: 좌표가 맞을 때만 클릭한다. 키 입력·명령 실행도 현재 포커스가 맞을 때만 한다.
7. **Verify**: 행동 직후 다시 캡처해 예상한 화면 변화가 생겼는지 확인한다.
8. **Troubleshoot**: 결과가 다르면 같은 행동을 반복하지 말고 다른 가설을 세운다. 모든 분기에 캡처 경로를 남긴다.

## 클릭 금지 조건 (하나라도 해당하면 클릭하지 말고 Observe로 돌아간다)
- 행동 전 캡처가 없다.
- 포인터 이동 후 캡처 확인이 없다.
- 목표 UI가 화면에 보이는지 불확실하다.
- 팝업·권한창·로그인창·다른 탭이 화면을 가리고 있다.
- OTP·URL·탭·포커스가 최신 상태인지 확인되지 않았다.
- 비전 모델이 제안한 좌표를 OCR/DOM/접근성/재캡처로 검증하지 않았다.

## 보안 불변식 (law.md 기준 — 원본의 보안 게이트는 우리 규약으로 재매핑)
원본 헌장의 보안성 규칙(로그인·OTP·자격증명·연결정보 취급)은 **번호로 된 외부 게이트를 그대로 옮기지 않고** 이 워크스페이스의 보안 불변식(law.md §7)으로 재매핑해 적용한다.
- **실제 비밀값을 본문에 평문으로 두지 않는다.** 계정·비밀번호·OTP·토큰·원격 연결정보(주소/키)는 이 SKILL.md나 cases.jsonl 같은 공개(`기본/`) 문서에 평문으로 적지 않는다.
- 개인정보·특정 회사/고객 정보·자격증명·내부 전용 연결정보가 필요한 작업은 그 자료를 **`대외비/` 등급**에 두고, 그 자원은 발견기로 찾아 참조한다(law.md §7·§8).
- 화면에 OTP·비밀번호·토큰이 캡처될 수 있으므로, 증거 스크린샷에 비밀값이 노출되면 공개 경로에 남기지 않는다(민감하면 대외비 처리).
- 실제 원격제어·인프라 구성(원격 데스크톱/VNC/터널/컨테이너 등)·VM 기동은 이 헌장 적용만으로 진행하지 않고, **해당 작업의 결재·범위 승인을 받은 뒤** 별도 스킬/도구로 수행한다.

## 스킬·지식 선택 (우리 발견 규약으로 재작성)
컴퓨터유즈 요청을 받으면, 특정 경로를 콕 박지 말고 **발견기(law.md §6)로 frontmatter description을 검색**해 지금 화면/앱에 맞는 스킬·지식을 찾는다.
1. 요청 키워드(앱 이름, 화면, 동작)로 관련 **스킬·지식 후보**를 발견기로 추린다.
2. 후보들의 description을 비교해 적합한 스킬의 `SKILL.md`를 읽고, 이 헌장의 루프 위에서 응용한다.
3. 컨텍스트가 부족하면 전체 파일을 길게 읽지 말고 검색 결과·핵심 절차·필요한 함수/명령만 읽는다.

## 실패 분기 예시
- **클릭 후 변화 없음**: 좌표 오류, 창 비활성, 다른 탭 활성, 투명 오버레이, 로딩 중을 의심한다.
- **입력 후 제출 안 됨**: `Enter`가 버튼을 누르지 못했거나 분리 입력칸의 이벤트가 다를 수 있으므로, 버튼 클릭 전후 캡처를 남긴다.
- **OTP 실패**: 오래된 메일, 캐시된 피드/탭, 만료 코드, 잘못된 스레드 선택을 의심한다.
- **Revit(앱) 실행 실패**: 프로세스 존재와 사용자에게 보이는 창은 다르므로, 프로세스/창핸들/화면 캡처를 분리해 판정한다.
- **브라우저 자동화 실패**: DOM 상태와 실제 사용자 화면이 다를 수 있으므로, CDP/DOM 결과만으로 완료 처리하지 않는다.

## 보고 기준
최종 보고에는 다음을 포함한다.
- 수행한 주요 캡처 경로
- 어떤 화면을 보고 어떤 판단을 했는지
- 어떤 행동을 했고 행동 후 화면이 어떻게 바뀌었는지
- 실패한 경우: 마지막 화면, 실패 가설, 다음 조치

**증거 캡처가 없으면 GUI 작업을 완료로 보고하지 않는다.**
