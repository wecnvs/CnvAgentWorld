# computer-use-charter (컴퓨터유즈 공통 헌장)

이 스킬은 어떤 엔진·에이전트든 컴퓨터 화면을 직접 조작(컴퓨터유즈, CU)할 때 **가장 먼저** 적용하는 공통 헌장이다. macOS/Windows CU, 브라우저 제어, Parallels/Windows VM, Autodesk/Revit 로그인, 대시보드 UI 조작, MCP(API) 결과의 시각 검수에 공통으로 적용한다. 앱별 세부 스킬(computer-use-mac/win, 원격-VNC-GUI 등)이 따로 있으면 이 헌장을 먼저 적용한 뒤 그 스킬을 발견기로 찾아 읽는다.

**한 줄 요지:** 화면 조작은 추측이 아니라 관찰이다 — `캡처→판단→이동→확인→행동→검증` 루프를 돌리고, 증거 캡처 없이 완료를 말하지 않는다.

## ⚠ 복잡도 안내
다음 중 하나라도 해당하면 요청자에게 상위 모델(예: Claude Opus / Sonnet) 사용을 권장한 뒤 진행/중단을 묻는다.
- [ ] 화면 변화·API 검수 결과가 비가역적이거나 중요한 모델 정합성에 영향을 준다.
- [ ] 좌표·타이밍이 민감한 다단계 GUI 흐름(로그인→OTP→설정 변경 등)을 한 번에 정합하게 맞춰야 한다.
- [ ] 여러 창/탭/세션(원격·VM 포함)에 걸쳐 상태 일관성을 유지해야 한다.

권장 안내 메시지(그대로 사용):
> "이 CU 작업은 정합성·관찰 부담이 큽니다. 저성능 모델은 오조작 위험이 있어 Opus/Sonnet 사용을 권장드립니다. 그래도 진행할까요?"

## 언제 쓰나
- 화면을 보고 클릭/타이핑해야 할 때(CU), 앱을 실행할 때, 브라우저를 조작할 때, 원격 데스크톱/VM 화면을 제어할 때.
- 로그인·OTP 입력, Revit 실행, 마우스/키보드 자동화 등 화면 변화를 관찰·검증해야 하는 모든 GUI 작업.
- **MCP(API) 자동화 결과 검수**: 코드로 구현하기 어려운 부분은 CU로 보완하고, 최종 검수는 API 성공 응답에만 의존하지 말고 **반드시 CU로 결과 뷰를 직접 캡처해** 시각적으로 확인한다.
- **도구 대체(브라우저 다운로드)**: 웹/클라우드(구글드라이브 등) 파일을 받아야 하는데 쉘·네트워크·전용 다운로드 도구가 없어 보일 때. '도구가 없다'며 되돌리지 말고 CU로 브라우저를 띄워 직접 받는다(아래 "클라우드 공유 폴더 일괄 다운로드").

## 원칙 (모든 엔진 공통)
화면 조작은 **추측 실행이 아니라 관찰 기반 작업**이다. 성능 수준과 무관하게 아래 표준 루프를 따른다. 경량 엔진(로컬 모델 등)도 순서를 생략하지 않는다 — 경량화는 허용하되 관찰 원칙 생략은 허용하지 않는다. **왜:** 화면은 언제든 팝업·로딩·포커스 이동·프레임 정지로 우리가 가정한 상태와 달라질 수 있어, 매 행동 전후로 눈(캡처)으로 실제 상태를 확인해야만 오조작·교착·헛클릭을 막는다.

## 표준 루프
1. **Lock**: 가능하면 CU 락(동시 조작 방지)을 먼저 잡는다. 못 잡으면 클릭·타이핑을 시작하지 않는다.
2. **Observe**: 행동 전 스크린샷을 저장하고 비전/OCR/DOM/프로세스로 현재 화면을 판정한다.
3. **Plan**: 목표 UI가 보이는지, 포커스가 맞는지, 팝업·로그인·로딩·만료·잠금 상태인지 가설을 세운다.
4. **Move**: 클릭이 필요하면 먼저 포인터만 목표 위로 이동한다.
5. **Confirm**: 이동 후 다시 캡처해 포인터가 목표 위인지 확인한다. 불확실하면 클릭하지 않는다.
6. **Act**: 좌표가 맞을 때만 클릭한다. 키 입력·명령도 포커스가 맞을 때만 한다.
7. **Verify**: 행동 직후 다시 캡처해 예상한 변화가 생겼는지 확인한다. API 연동도 최종엔 뷰포트를 캡처해 시각 검증한다.
8. **Troubleshoot**: 결과가 다르면 같은 행동을 반복하지 말고 다른 가설을 세운다. 모든 분기에 캡처 경로를 남긴다.

## 클릭 금지 조건 (하나라도 해당하면 클릭하지 말고 Observe로 돌아간다)
- 행동 전 캡처가 없다.
- 포인터 이동 후 캡처 확인이 없다.
- 목표 UI가 화면에 보이는지 불확실하다.
- OTP·URL·탭·포커스가 최신 상태인지 확인되지 않았다.
- 비전 모델이 제안한 좌표를 OCR/DOM/접근성/재캡처로 검증하지 않았다.

## 자연어 → CU 도구 입력 변환 (에이전트가 수행)
이 헌장은 특정 JSON을 실행 스크립트에 넘기는 실행형 스킬이 아니라 **모든 CU 위에 얹는 규약**이다. 그래도 하위 CU 도구(cu-mac/cu-win의 screenshot·click·type 등)를 부를 땐 사용자 말을 아래처럼 도구 입력으로 정규화한다.

| 사용자가 한 말 | CU 도구로의 변환 (예시) |
|----|----|
| "Revit 창 캡처해줘" | `screenshot` 실행 → 캡처본을 비전으로 분석 |
| "결과 확인해줘" | API 응답 확인 + **반드시** `screenshot`으로 결과 뷰를 물리 캡처 |
| "이 버튼 눌러줘" | `screenshot` → 포인터 `move` → 재`screenshot` 확인 → `click` → 재`screenshot` 검증 |

**기본값**: 단위·좌표 미명시 → 먼저 캡처해 화면에서 좌표를 읽어 결정한다(추정 좌표로 바로 클릭 금지).

## 팝업·모달 처리 — 먼저 '이게 내 작업 경로인가'로 나눈다 (핵심)
CU 중 화면에 팝업·모달이 뜨는 건 흔하다. 여기서 **가장 흔한 두 실패**는 — (A) 작업에 필요한 권한 팝업을 사람이 눌러주길 기다리며 0진행, (B) 작업과 **무관한** 모달(예: 키체인/Chrome Safe Storage 암호 입력창)에 매달려 deny·escape만 반복하며 수 분을 허비 — 이다. 둘 다 막는 규칙은 하나다: **뜬 팝업이 '내 작업 경로 위'인지부터 판정하고 분기한다.**

### 1) 먼저 캡처로 팝업을 읽고 분류한다
행동 전후 캡처는 항상 있어야 한다(표준 루프). 캡처에서 팝업 문구·버튼을 읽어 **작업관련**인지 **무관**인지 판정한다.
- **작업관련(경로 위)** = 이걸 넘겨야 내 다음 단계가 진행된다. 예: "'터미널'이 'Chrome'을 제어하려 함", "다운로드 폴더 접근 허용", 로그인 진행에 필요한 확인창.
- **무관(경로 밖)** = 내 작업과 상관없이 우연히 뜬 것. 예: 지금 작업에 필요 없는 키체인/Safe Storage 암호 입력창, OS 업데이트 알림, 딴 앱 알림.

### 2) 작업관련 팝업 → 네가 직접 처리한다 (사람·관리자 대기 금지)
사람이 대신 눌러주길 기다리지 마라 — 너는 손쉬운 사용 권한이 있어 클릭이 실제로 먹는다.
- ① 캡처로 버튼 위치를 잡고 ② **'항상 허용'(있으면 우선) 또는 '허용'**을 클릭해 넘긴다('항상 허용'은 영구 부여라 다음부턴 안 뜬다) ③ 재캡처로 팝업이 사라졌는지 확인하고 원래 작업을 잇는다.
- **키보드는 시스템 팝업에 안 먹는다 — 반드시 클릭으로.** Return/Enter로 닫으려 하지 마라.
- **애초에 이 팝업을 만들지 마라:** 브라우저·앱 조작을 `osascript`(`tell application …`)로 하지 마라 — 그건 '앱 제어' 자동화 팝업을 띄우고 **그 osascript 호출 안에서 응답을 기다리며 교착**된다(자기 손도 갇힌다). 화면 UI(주소창·버튼·메뉴)는 **cu 도구의 캡처+클릭으로 직접** 누른다. 앱 실행/전면화만 필요하면 셸 `open -a "앱이름"`.

### 3) 무관 모달 → 한 번만 닫아보고, 안 되면 '막힘' 보고 후 다음 단계 (시간 허비 금지)
무관 모달에 deny·escape를 반복하며 매달리지 마라. **1회만** 닫아보고(작업 창을 가리지 않으면 그냥 두고 곧바로 본 작업을 잇는다), 1~2회 시도해도 안 닫히면 거기 붙들지 말고 **'이 모달 때문에 막힘'이라고 정확히 보고한 뒤 다음 단계로 넘어간다.** 무관 모달 하나에 수 분을 쓰지 않는다. **왜:** 무관 모달은 내 작업의 완료 조건이 아니다 — 그걸 없애는 게 목표가 아니라 내 작업 단계를 진행하는 게 목표다. 화면을 실제로 가리지 않는 한 그것은 '장애물'이 아니라 '배경'이다.

### 4) 예외 — 비밀번호를 새로 '입력'해야만 하는 경우
'항상 허용/허용'으로 안 넘어가고 **암호를 타이핑해야만** 열리는 창(로그인 키체인 새 암호 등)은, 비밀번호를 지어내거나 입력하지 말고 그 사실을 보고하고 멈춘다(사람 1회 조치 필요). 그 창이 작업과 무관하면 3)대로 두고 진행한다.

## 보안 불변식 (law.md §7 재매핑 — 원본의 번호 게이트를 우리 규약으로)
- **실제 비밀값을 본문·로그·캡처에 평문으로 두지 않는다.** 계정·비밀번호·OTP·토큰·원격 연결정보(주소/키)는 이 SKILL.md·cases.jsonl 같은 공개(`기본/`) 문서나 도구 인수·로그에 평문으로 남기지 않는다.
- 개인정보·특정 회사/고객 정보·자격증명·내부 연결정보가 필요한 작업은 그 자료를 **`대외비/` 등급**에 두고 발견기로 참조한다(law.md §7·§8).
- 화면에 OTP·비밀번호·토큰이 캡처될 수 있으므로, 비밀값이 노출된 증거 스크린샷은 공개 경로에 남기지 않는다(민감하면 대외비 처리).
- 실제 원격제어·인프라 구성(원격 데스크톱/VNC/터널/컨테이너)·VM 기동은 이 헌장 적용만으로 진행하지 않고, **결재·범위 승인을 받은 뒤** 별도 스킬/도구로 수행한다.

## non_overridable
- 비밀값(계정·비밀번호·OTP·토큰·연결정보)을 공개 경로에 평문으로 남기지 않는다.
- 증거 캡처가 없으면 GUI 작업을 완료로 보고하지 않는다.
- 행동 전 캡처와 포인터 위치 확인 없이 추측으로 클릭하지 않는다.
- 작업과 무관한 모달에 수 분을 허비하지 않는다 — 1~2회로 안 되면 막힘을 보고하고 다음 단계로 진행한다.

## 스킬·지식 선택 (발견 규약)
CU 요청을 받으면 특정 경로를 콕 박지 말고 **발견기(law.md §6)로 frontmatter description을 검색**해 지금 화면/앱에 맞는 스킬·지식을 찾는다.
1. 요청 키워드(앱 이름, 화면, 동작)로 관련 스킬·지식 후보를 발견기로 추린다(예: 맥→computer-use-mac, 윈도우/Revit→computer-use-win, 원격→원격-VNC-GUI).
2. 후보 description을 비교해 적합한 스킬의 `SKILL.md`를 읽고, 이 헌장 루프 위에서 응용한다.
3. 컨텍스트가 부족하면 전체 파일을 길게 읽지 말고 검색 결과·핵심 절차·필요한 명령만 읽는다.

## 클라우드 공유 폴더 일괄 다운로드 — 조기 진단·우회 (같은 벽에 재시도 금지)
구글드라이브 등 클라우드 공유 폴더를 **일괄** 받을 때, 파일마다 접근 조건이 다르다. 어떤 파일은 '링크가 있는 모든 사용자'로 공개, 어떤 파일은 '링크 공유 아님'(특정 계정 공유)이라 비인증 도구로 불가, 어떤 파일은 다운로드 도구의 **파일당 크기 상한**(예: Drive MCP 10MB) 초과라 그 도구로 불가하다. 같은 파일을 같은 수단으로 반복 호출하면 **타임아웃만 쌓인다.** 관찰 원칙을 다운로드에도 적용한다 — **받기 전에 '무엇을·어떤 수단으로 받을 수 있는지'를 먼저 진단**하고, 벽은 재시도가 아니라 우회·에스컬레이션으로 넘는다.

### 절차 (결정 트리)
1. **먼저 진단(인벤토리).** 폴더 파일 목록을 훑어 파일별 (a) 공유 상태와 (b) 크기를 먼저 확보한다(메타데이터 먼저, 그 위에 수단 배정).
2. **수단별 벽을 안다.**

| 다운로드 수단 | 인증 | 받을 수 있는 파일 | 벽(한계) |
|----|----|----|----|
| gdown (비인증) | 없음 | '링크가 있는 모든 사용자' 공유 파일만 | 권한 제한(특정 계정) 파일 불가 |
| Drive MCP `download_file_content` | 계정 인증 | 인증 계정이 접근 가능한 파일 | 파일당 크기 상한(예: 10MB) 초과 불가 |
| computer-use 브라우저 | 로그인 세션 | 로그인 계정이 접근 가능한 파일 | 대량엔 느림. 단 비인증·크기 벽은 로그인 세션이 있으면 우회 가능 |

> 구체 수치·도구명은 환경마다 다르다. 요지는 **"수단마다 인증/크기 벽이 다르니 파일의 (공유상태·크기)에 수단을 맞춘다"**는 것.

3. **획득 매트릭스로 분류하고 받을 수 있는 것부터 전량 확보.** 공개(링크 공유)→gdown / (권한제한+상한 이하)→인증 수단(Drive MCP 등) / (권한제한+상한 초과, 대용량)→로그인 세션이 있으면 CU 브라우저 우회, 없으면 5)로.
4. **인벤토리로 문서화.** 획득/미획득을 `파일명 · 링크 · 크기 · 사용 수단 · (미획득) 사유`로 표에 남긴다 — 진행 증적이자 재획득 요청의 근거 명세.
5. **막힌 것은 재시도 말고 에스컬레이션.** '권한제한 + 대용량'처럼 현재 수단으로 불가한 파일은 같은 벽에 재시도하지 말고 요청자에게 (1) '링크가 있는 모든 사용자'로 공유 변경, 또는 (2) Drive API 토큰(인증 자격) 제공 중 하나를 요청한다. 충족되면 **막혔던 파일만** 재획득하고 인벤토리를 갱신한다.

## 안티패턴 (하지 마라)
- ❌ 행동 전후 캡처 없이 추정 좌표로 바로 클릭 → 헛클릭·오조작. Observe로 돌아가라.
- ❌ 클릭 후 변화가 없는데 같은 행동 반복 → 좌표 오류·창 비활성·다른 탭·투명 오버레이·로딩 중을 의심하고 재캡처로 재판정.
- ❌ 작업관련 권한 팝업을 사람이 눌러주길 대기 → 네가 직접 '허용'을 클릭한다(팝업 처리 2절).
- ❌ 무관 모달에 deny·escape를 반복하며 수 분 허비 → 1~2회로 안 되면 막힘 보고 후 다음 단계(팝업 처리 3절).
- ❌ 브라우저를 osascript로 원격 제어 → 자동화 팝업 교착. cu 캡처+클릭으로 직접.
- ❌ API 성공 응답·DOM 상태만 보고 완료 처리 → 최종엔 뷰포트 캡처로 시각 검증.
- ❌ 클라우드 일괄 다운로드에서 파일별 진단 없이 한 수단으로 폴더 전체 반복 → 권한/상한 벽에서 타임아웃 무한 반복.
- ❌ 일부 실패를 이유로 "다운로드 불가"라며 전체 되돌림 → 받을 수 있는 건 받고, 못 받는 것만 사유·재획득 경로와 함께 남긴다(부분 성공 + 명확한 차단 사유).

## 실패 분기 예시 (증상 → 의심)
- **클릭 후 변화 없음**: 좌표 오류, 창 비활성, 다른 탭 활성, 투명 오버레이, 로딩 중.
- **입력 후 제출 안 됨**: `Enter`가 버튼을 못 눌렀거나 분리 입력칸의 이벤트가 다름 — 버튼 클릭 전후 캡처를 남긴다.
- **OTP 실패**: 오래된 메일, 캐시된 피드/탭, 만료 코드, 잘못된 스레드 선택.
- **앱(Revit) 실행 판정**: 프로세스 존재와 사용자에게 보이는 창은 다름 — 프로세스/창핸들/화면 캡처를 분리해 판정.
- **원격/분리 세션 화면 정지(frozen)**: '앱이 안 뜬다'고 오진하지 말 것 — 프레임버퍼 정지일 수 있다. 프로세스로 기동 확인 + 창 직접 캡처로 우회(세부는 computer-use-win을 발견해 적용).
- **브라우저 자동화**: CDP/DOM 결과만으로 완료 처리하지 않는다 — 실제 사용자 화면과 다를 수 있다.

## 보고 기준
최종 보고에 다음을 포함한다.
- 수행한 주요 캡처 경로
- 어떤 화면을 보고 어떤 판단을 했는지
- 어떤 행동을 했고 행동 후 화면이 어떻게 바뀌었는지
- 막힘·실패: 마지막 화면, 실패 가설, 다음 조치(무관 모달로 막혔으면 그 사실도 명시)

**증거 캡처가 없으면 GUI 작업을 완료로 보고하지 않는다.**

## 완료 및 검증 기준 (DoD)
이 헌장을 적용한 CU 작업의 완료(Done) 조건과 물리 검증 체크리스트:
- **[DoD 1] 행동 전후 캡처**: 조작 전(목표 확인)·이동 후(포인터 위치)·행동 직후(결과)의 스크린샷이 워크스페이스 내 물리 이미지 파일로 저장되어 있어야 한다.
- **[DoD 2] API 검수 시각화**: MCP/API 결과 검수 시, 성공 응답 텍스트만이 아니라 CU로 결과 화면을 직접 캡처한 이미지가 증적으로 있어야 한다.
- **[DoD 3] 실패/성공 로그**: 실패 가설·성공 판정에 쓰인 캡처 경로와 판단 근거가 최종 보고에 누락 없이 명문화되어야 한다.
- **[DoD 4] 팝업 처리 증적**: 작업관련 팝업을 직접 처리했으면 처리 전후 캡처가, 무관 모달로 막혔으면 '막힘' 보고가 남아 있어야 한다.

## 케이스 누적
이 헌장의 원칙·루프·불변식은 안정 본문이다. 실제 적용에서 얻은 경우의 수·교훈은 `cases.jsonl`(CaseLedger.v1)에 누적한다 — 본문을 매번 손대지 않고 케이스로 성장하며, 여러 케이스로 수렴한 보편 교훈만 이 본문에 녹인다(단일 특수 케이스로 본문 고정 금지).
