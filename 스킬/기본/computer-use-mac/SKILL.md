# computer-use-mac (맥 컴퓨터유즈)

macOS에서 에이전트가 **직접 눈(스크린샷)과 손(마우스/키보드)** 이 되어 GUI를 자동화하는 스킬이다. 별도 API 호출 없이, vision+bash가 되는 엔진이 화면을 읽고 판단해 도구를 호출한다.

이 본문은 **불변(절차·원칙)** 이다. 실제 적용에서 얻은 경우의 수·실전 교훈(패러럴즈 좌표 보정, 스크롤 미수신, frozen 창 캡처 우회 등)은 본문에 쌓지 않고 `cases.jsonl`(CaseLedger.v1)에 누적한다.

## 먼저 할 것 — 공통 헌장 적용
컴퓨터유즈는 앱별 스킬보다 **공통 헌장이 우선**이다. 이 스킬을 쓰기 전에 `컴퓨터유즈 공통 헌장(캡처→판단→이동→확인→행동→검증, 클릭 금지 조건, 보안 불변식)`을 **발견기로 찾아 먼저 적용**하고(현재 워크스페이스의 `computer-use-charter`), 그 루프 위에서 이 스킬의 mac 도구를 호출한다.

## 언제 쓰나
- macOS 화면을 보고 클릭/타이핑/스크롤해야 할 때, 앱을 실행할 때, 브라우저를 조작할 때.
- 스크린샷으로 현재 상태를 관찰·검증하며 GUI 작업을 단계적으로 진행해야 할 때.
- (원격 데스크톱/VM/Windows 게스트 등 다른 환경은 각 환경의 스킬을 발견기로 찾아 함께 쓴다 — 이 스킬은 mac 호스트 조작이 범위다.)

## 핵심 아키텍처
**에이전트가 직접 눈이 되고 손이 된다.** 별도 API 호출 없음.
```
에이전트(vision+bash) ── [눈] screenshot.py → 이미지 읽기로 화면 판정
                      ── [판단] 목표 UI 위치·상태 추론, log.py로 기록
                      ── [손] move→screenshot 확인→click/type/scroll
                      └─ 결과를 screenshot으로 재확인하며 반복
```

## 도구는 '호출만' 한다 (구현 인라인 금지)
이 스킬은 도구를 **호출만** 한다. pyautogui 등으로 직접 코드를 짜지 않는다. 실제 구현은 도구 묶음 **cu-mac** 에 분리되어 있다 — `컴퓨터유즈용 mac 화면 제어 원자 도구(스크린샷·클릭·입력·좌표조회·락)` 를 발견기로 찾아(현재 `도구/기본/cu-mac/`) 그 `run_tool.sh` 래퍼로 호출한다.

```bash
bash 도구/기본/cu-mac/run_tool.sh <script.py> [인수...]
```

호출 인터페이스(구현은 cu-mac 도구.md/소스 참조):

| 목적 | 호출 |
|---|---|
| 화면 캡처(포인터 빨강 마커, HUD 제외) | `run_tool.sh screenshot.py /tmp/cu_screen.png` |
| 포인터 이동만(클릭 X) | `run_tool.sh move.py <x> <y>` |
| 클릭(좌/우/더블) | `run_tool.sh click.py <x> <y> [--right|--double]` |
| 텍스트 입력(한국어 포함) | `run_tool.sh type_text.py "텍스트"` |
| 키/단축키 | `run_tool.sh press_key.py "command+space"` |
| 스크롤 | `run_tool.sh scroll.py <x> <y> down [n]` |
| 창/주소창 좌표 조회(접근성) | `run_tool.sh find_element.py windows [App]` |
| 브라우저 요소 정확좌표(CDP) | `run_tool.sh web_locate.py "버튼 텍스트"` |
| HUD 로그/판단/완료 | `run_tool.sh log.py "메시지"` (`--think`/`--done`) |
| HUD 오버레이 시작/중지 | `bash 도구/기본/cu-mac/overlay_start.sh [start|stop|status]` |
| 동시조작 방지 락 | `run_tool.sh cu_lock.py acquire <agent_id> --note "작업"` (heartbeat/release/status) |

## 표준 실행 루프 (헌장 루프의 mac 구체화)
1. **락**: GUI 조작 전 `cu_lock.py acquire <agent_id>` 로 컴퓨터유즈 락을 잡는다(이 머신은 물리 화면 1개라 동시 조작 시 충돌). 실패면 시작하지 않고 대기/보고. 긴 작업은 `heartbeat`로 연장, 끝나면 반드시 `release`.
2. **선언/HUD**: 필요 시 `overlay_start.sh start` 후 `log.py "🚀 작업 시작: <요청 한 줄>"`.
3. **관찰**: `screenshot.py` → 이미지 읽기로 현재 화면 판정. `log.py --think "상태+다음 행동"`.
4. **좌표 확정**: 작은 버튼·웹 요소는 눈대중 추측 금지 — `find_element.py`/`web_locate.py`로 정확 좌표를 조회.
5. **이동→확인**: `move.py x y` 후 `screenshot.py`로 빨강 포인터 마커가 목표 위인지 눈으로 확인. 빗나가면 보정 후 재시도.
6. **행동**: 맞을 때만 `click.py`/`type_text.py`/`press_key.py`/`scroll.py`.
7. **검증**: 행동 직후 `screenshot.py`로 예상한 변화를 확인. 다르면 같은 행동을 반복하지 말고 다른 가설(좌표·포커스·팝업·로딩)을 세운다.
8. **완료**: 최종 결과를 `screenshot.py`로 시각 검증한 뒤 `log.py --done "완료 요약(시각 확인)"`. 증거 캡처 없이 완료 보고 금지.

## 클릭 금지 조건 (하나라도 해당하면 클릭하지 말고 관찰로 복귀)
- 행동 전 캡처가 없다 / 포인터 이동 후 캡처 확인이 없다.
- 목표 UI가 화면에 보이는지 불확실하다.
- 권한 팝업·로그인창·다른 탭·오버레이가 화면을 가리고 있다.
- 비전이 제안한 좌표를 재캡처/접근성/CDP로 검증하지 않았다.

## 보안 불변식 (law.md §7로 재매핑)
원본 스킬의 보안 규칙은 외부 게이트 번호를 그대로 옮기지 않고 이 워크스페이스의 보안 불변식으로 재매핑해 적용한다.
- **비밀값을 평문으로 두지 않는다.** 계정·비밀번호·OTP·토큰·원격 연결정보(주소/키)를 이 SKILL.md·`cases.jsonl`·도구 인수·로그·캡처 등 공개(`기본/`) 경로에 평문으로 남기지 않는다.
- 개인정보·특정 회사/고객 정보·자격증명·내부 전용 연결정보가 필요한 작업은 그 자료를 **`대외비/` 등급**에 두고 발견기로 참조한다.
- OTP·비밀번호·토큰이 찍힌 증거 스크린샷은 공개 경로에 남기지 않는다(민감하면 대외비 처리).
- 실제 원격제어·인프라 구성(원격 데스크톱/VNC/터널/컨테이너)·VM 기동은 이 스킬 적용만으로 진행하지 않고, **해당 작업의 결재·범위 승인** 후 별도 스킬/도구로 수행한다.

## 권한·환경 (처음 1회)
- 손쉬운 사용(Accessibility)·화면 녹화 권한을 에이전트가 실행 중인 터미널 앱에 부여해야 클릭/캡처가 동작한다.
- 환경 점검·의존성 확인이 필요하면 cu-mac 묶음의 점검 절차(또는 원본 `scripts/`)를 발견기로 찾아 1회 실행한다.

## 보고 기준
주요 캡처 경로, 무엇을 보고 어떻게 판단했는지, 행동 전후 화면 변화, (실패 시) 마지막 화면·실패 가설·다음 조치를 함께 보고한다. **증거 캡처가 없으면 완료로 보고하지 않는다.**
