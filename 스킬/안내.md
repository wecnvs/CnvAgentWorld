# 스킬 (Skills)

**도구들을 조합한 방법론.** 예: '계산기' — 곱셈·뺄셈 같은 기본 [[도구]]들을 엮어
하나의 산식을 푸는 *절차/방법*을 담은 것. 도구가 부품이라면 스킬은 그 부품들로 만든 조립품이다.

## 하위 구성 (모든 자원 폴더 공통)
- **기본** — 워크스페이스 구동에 *반드시 필요한* 것. → **Git 추적(깃허브에 올라감).**
- **추가** — 일반 확장. → 로컬 전용(Git 제외).
- **고급** — 고급·특수 용도. → 로컬 전용(Git 제외).
- **대외비** — 민감·비공개. → 로컬 전용(Git 제외).

## 스킬 규약 (필수)

스킬은 **사용**한다 — 단 100% 복제가 아니라 참고해 적재적소로 변형·활용한다(law.md §6).

### 폴더 구조
```
스킬/<등급>/<skill-name>/
├── SKILL.md     (필수 — 매니페스트 + 본문)
├── scripts/     (이 스킬 전용 고정 실행 코드)
├── references/  (참고용 코드/문서)
└── assets/      (산출물용 파일)
```
> 범용 기능은 스킬에 박지 말고 [[도구]]로 만든다(`스킬/기본/skill-creator`의 "도구 우선" 절).

### 매니페스트 SKILL.md (frontmatter)
`name` + `description`(*무엇을 + 언제 쓰는지 + 사용자 표현 예시 + 핵심 용어* — 발견 최적화).

### 발견 (태그 없음)
- 스킬은 **frontmatter `description`으로 발견**된다. 등록·태그가 필요 없다 — 발견기가 매번 스캔.
- `description` 품질이 곧 발견 품질이다. 또렷이 쓴다.

> 스킬 생성·개선의 전체 절차는 `스킬/기본/skill-creator`. 무엇을 쓸지 찾는 건 `스킬/기본/발견기`.
