# 지식 (Knowledge)

**생각하거나 작업할 때 참고하는 내용.** 판단·작업의 근거가 되는 참고자료·정리·맥락.
[[도구]]·[[스킬]]이 '하는 법'이라면, 지식은 '참고할 내용'이다.

## 하위 구성 (모든 자원 폴더 공통)
- **기본** — 워크스페이스 구동에 *반드시 필요한* 것. → **Git 추적(깃허브에 올라감).**
- **추가** — 일반 확장. → 로컬 전용(Git 제외).
- **고급** — 고급·특수 용도. → 로컬 전용(Git 제외).
- **대외비** — 민감·비공개. → 로컬 전용(Git 제외).

## 지식 만들기 규약 (필수)

지식은 **참고**용이다(스킬은 사용, 지식은 참고 — law.md §6).

### 폴더 구조
```
지식/<등급>/<knowledge-name>/
├── 지식.md     (필수 — 매니페스트 + 본문)
└── <첨부 자료>  (선택)
```

### 매니페스트 `지식.md` (frontmatter)
```yaml
---
name: <knowledge-name>
description: <무엇에 대한 참고 지식인가 + 언제 참고하나 + 핵심 용어>   # 발견기가 잘 찾게
---
```

### 발견 (태그 없음)
- 지식은 **frontmatter `description`으로 발견**된다. 등록·태그가 필요 없다 — 발견기가 매번 스캔.
- `description`을 또렷이 써야 잘 찾힌다.

### 본문에서 도구·자산·다른 지식 참조 (law.md §8)
자연어로 적고 발견기로 찾는다: `관련 도구: 두 수 곱하기 — 발견기로 찾아 사용`, `참고 지식: 배포 규칙 — 발견기로 찾아 참고`.

> 지식을 만들고 정리하는 자세한 절차는 [[스킬]] `스킬/기본/knowledge-creator`.
