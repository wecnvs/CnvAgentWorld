# 앱 (Apps)

**우리가 만들어 쓰거나 가져다 쓰는 별도 애플리케이션을 등록·관리하는 곳.** 한 앱 = 한 폴더이고,
그 폴더 안에 **소스코드 · 설치파일 · 실행기**가 함께 들어간다(개발·배포·실행이 한 폴더에서 관리됨).
대시보드의 **뷰어 → 앱 탭**이 이 폴더를 라이브 스캔해 종류에 맞는 버튼(실행/열기/설치파일 다운로드/중지)으로 보여준다.

> 앱을 **등록·개발·세팅·반입**할 때는 스킬 **`스킬/기본/앱등록`**을 따른다("앱 만들자/등록하자/개발하자/블렌더 가져오자" 등).

## 앱의 종류(kind)와 동작
| kind | 무엇 | 대시보드 동작 |
|------|------|----------------|
| `web-app` | 우리가 만든 웹 애플리케이션 | **실행** = 서버 컴퓨터에서 `port`를 열고, 내가 접속한(예: Tailscale) 주소로 **내 브라우저**가 그 포트를 새 탭으로 연다. **중지**로 끈다. |
| `standalone` | 우리가 만든 독립실행형(서버에서 도는 것) | **실행** = 서버 컴퓨터에서 실행기를 띄운다(에이전트가 MCP 등으로 그 프로그램을 사용). **중지** 가능. |
| `external` | 외부 프로그램(예: Blender) — 소스 없음, 설치/실행파일만 | **실행** = 서버 컴퓨터에서 띄운다. **중지** 가능. |
| `revit-addin` / `install-only` | Revit 애드인처럼 설치해서 쓰는 것 | **설치파일 다운로드**만(실행 없음). 받아서 해당 프로그램에 설치한다. |

핵심: **web-app은 포트를 열어 내 브라우저로, 나머지 실행형은 서버 호스트에서, 설치형은 다운로드만.**

## 실행 위치(target) — 서버 컴퓨터를 중앙 컨트롤센터로
앱은 **서버 컴퓨터뿐 아니라 네트워크의 원격 윈도우·VM에서도** 실행할 수 있다. 앱.md의 `target`으로 *어디서 실행되는지*를 정하면, 대시보드가 **카드에 실행 위치 배지**(🖥️서버 / 🪟원격·VM …)로 구분해 보여주고 그 자리에서 실행·중지한다.

| target 채널 | 어디서 | 디스패치 | 상태/중지 추적 |
|---|---|---|---|
| `local`(기본) | 이 서버 컴퓨터 | subprocess (Win/mac/Linux) | 완전 |
| `cu-helper` | 원격 윈도우/VM의 `cu_helper.ps1` 데몬 | HTTP `/run·/ps·/stop` | **완전**(레빗 등 GUI앱 권장 경로) |
| `parallels` | 이 Mac의 Parallels 게스트 | `prlctl exec <vm> <cmd>` | 제한(런처만) |
| `ssh` | OpenSSH 있는 원격 | `ssh <ssh> <cmd>` | 제한(런처만) |

- `target`을 안 적으면 `local`(서버). 원격/VM은 별칭으로 적고, 그 별칭→연결정보는 **타깃 레지스트리**에 둔다.
- **타깃 레지스트리(대외비)**: `자산/대외비/앱실행대상/targets.json` — `{"targets":[{"name","channel","host","port"|"vm"|"ssh","label"}]}`. 실제 host/IP·VM명·자격증명은 **대외비**(law.md §7) — 공개 경로·로그·매니페스트에 평문 금지. 스키마는 같은 폴더 `targets.example.json`.
- **원격 윈도우에서 레빗 실행(대표 시나리오)**: 그 윈도우에 `cu_helper.ps1`을 대화형 세션에서 띄우고(`도구/기본/vm-cu-helper`), 레지스트리에 `cu-helper` 타깃으로 등록 → 앱.md `target: <그 별칭>`, `run: "C:\…\Revit.exe"`. 앱 탭의 '실행'이 그 윈도우에서 레빗을 띄우고 '중지'로 끈다. 띄운 레빗은 에이전트가 `computer-use-win`/`vnc-mcp-원격실행`으로 조작.
- 원격 `cu-helper`는 GUI앱까지 완전 추적(실행 중·중지). `ssh`/`parallels`는 런처만 추적돼 중지가 원격 프로세스까지 보장되진 않는다(배지에 '추적 제한' 표시).

## 하위 구성 (모든 자원 폴더 공통 등급)
- **기본** — 워크스페이스 구동에 *반드시 필요한* 가벼운 공개 앱. → **Git 추적.** (큰 바이너리·사내용 금지)
- **추가** — 일반 확장. → 로컬 전용(Git 제외).
- **고급** — 고급·특수 용도. → 로컬 전용(Git 제외).
- **대외비** — 회사/고객·내부 전용(사내 레빗 애드인·블렌더 등 외부 바이너리 다수). → 로컬 전용(Git 제외).

> **큰 바이너리·외부 프로그램·사내용은 `기본`에 두지 않는다.** 설치파일(.msi·.exe·.dmg)·외부 실행기는
> 저장소를 무겁게/민감하게 만든다 → **`추가`/`고급`/`대외비`(로컬 전용)** 에. (law.md §7 보안 불변식)

## 앱 폴더 표준 구조
```
앱/<등급>/<app-name>/
├── 앱.md          (필수 — 매니페스트. frontmatter로 발견·표시·동작 결정)
├── 소스/           (우리가 개발한 소스코드. 외부 프로그램이면 없음)
├── 설치/           (배포·설치파일 — '설치파일 다운로드' 대상: Setup.msi, *.addin …)
├── 실행/           (실행기·실행파일 — 'run' 대상: MyApp.app, run.sh, 런처 …)
└── .run/           (런타임 pid·port — 시스템이 생성, Git 제외. 손대지 않는다)
```
폴더를 다 채울 필요는 없다 — 종류에 맞는 것만 둔다(web-app=소스+실행, external=설치/실행, revit-addin=설치).

## 매니페스트 `앱.md` (frontmatter)
```yaml
---
name: <app-name>
description: <무엇을 하는 앱 + 언제 쓰나 + 핵심 용어>   # 앱 탭/발견이 잘 찾게
kind: web-app | standalone | external | revit-addin | install-only
platform: win | mac | cross | web     # (선택)
version: 0.1.0                        # (선택)
status: ready | wip | deprecated      # (선택)

# web-app 전용 ────────────────────────────────
port: 8090                            # 서버가 열 포트(내 브라우저가 이 포트로 접속)
run: python3 -m http.server 8090 --bind 0.0.0.0   # 그 포트를 0.0.0.0에 여는 서버 시작 명령

# standalone / external 전용 ──────────────────
run: open -a Blender                  # 서버 호스트에서 실행할 명령(cwd = 이 앱 폴더)

# 설치/다운로드 (어느 종류든) ──────────────────
install: 설치/Setup.msi               # '설치파일 다운로드' 버튼(앱 폴더 기준 경로)
download: 실행/MyApp.app               # 보조 다운로드 버튼
---

# 사용법 / 설치법 / 주의
설치 절차·의존성·주의사항. (앱 카드에서 일부 미리보기.)
```

## 실행(run)·열기 동작 자세히
- **web-app**: '실행'을 누르면 서버 컴퓨터에서 `run`이 돌아 `port`가 열린다. 그다음 대시보드가 **`http://<내가-접속한-호스트>:<port>`** 를 새 탭으로 연다.
  - 내가 Tailscale로 다른 컴퓨터에서 대시보드를 보고 있으면, `<내가-접속한-호스트>`가 그 Tailscale 주소라 **그 컴퓨터 브라우저에서 바로 뜬다.**
  - 그래서 web-app 서버는 **반드시 `0.0.0.0`(전체 인터페이스)에 바인드**해야 외부(Tailscale 피어)에서 닿는다. `localhost`/`127.0.0.1`로 바인드하면 다른 컴퓨터에서 안 열린다.
- **standalone / external**: '실행'은 `run`을 **서버 호스트에서** 돌린다(GUI는 서버 화면에 뜸). 목적은 에이전트가 그 프로그램을 MCP 등으로 쓰는 것.
- **중지**: 실행형 앱은 카드에 '중지'가 떠서 띄운 프로세스를 끈다(프로세스 그룹째 종료).
- **안전**: 대시보드는 **매니페스트의 `run`만** 돌린다(클라이언트가 임의 명령을 못 넣는다). `run`의 cwd는 그 앱 폴더. 절대경로 하드코딩 금지, 앱 폴더 기준으로 적는다.

## 표시·관리 (태그 없음 — 발견 철학과 동일)
- 앱은 **대시보드 앱 탭**이 `앱/`을 매번 라이브 스캔해 `앱.md` frontmatter로 표시·관리한다(등록·태그·수동 색인 없음 — law.md §7 발견 철학과 같다).
- `description`은 *무엇을 하나 + 언제 쓰나 + 핵심 용어*를 담아 알아보기 쉽게 쓴다.
- 앱은 사람이 쓰는 **배포 애플리케이션**이라 능력-발견기(`도구/기본/발견기`)의 스킬·지식·도구·자산 후보에는 들어가지 않는다. 앱의 발견 창구는 앱 탭이다.

> 경로는 모두 **루트폴더 기준**이다. 절대경로를 하드코딩하지 않는다(law.md §0).
