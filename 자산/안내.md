# 자산 (Assets)

**계속 쌓아두고 관리하는 것들.** 아이디(ID), API 키, 자료 등 — 워크스페이스가
보유·운용하는 자원. 특히 민감한 것은 `대외비`에 두어 Git에 올라가지 않게 한다.

## 하위 구성 (모든 자원 폴더 공통)
- **기본** — 워크스페이스 구동에 *반드시 필요한* 것. → **Git 추적(깃허브에 올라감).**
- **추가** — 일반 확장. → 로컬 전용(Git 제외).
- **고급** — 고급·특수 용도. → 로컬 전용(Git 제외).
- **대외비** — 민감·비공개(API키·자격증명 등). → 로컬 전용(Git 제외).

## 자산 만들기 규약 (필수)

자산은 스킬·지식이 필요할 때 **참조**하는 보유 자원이다(law.md §8).

### 폴더 구조
```
자산/<등급>/<asset-name>/
├── 자산.md     (필수 — 매니페스트. 비밀 '값'은 적지 않는다)
└── <실제 자산>  (값/데이터. 비밀이면 <등급>=대외비 — 폴더째 Git 제외)
```

### 매니페스트 `자산.md` (frontmatter)
```yaml
---
name: <asset-name>
description: <무슨 자산인가 + 언제 쓰나 + 핵심 용어 — 비밀 값은 쓰지 않는다>   # 발견기가 잘 찾게
---
```

### 비밀 취급 (가장 중요)
- API키·자격증명 등 비밀 '값'은 `자산/대외비/<이름>/`에 둔다(폴더째 gitignore). 코드는 런타임에 상대경로·환경변수로만 읽는다(절대경로 하드코딩 금지).
- `자산.md`엔 **설명만** 적고 **비밀 값은 절대 적지 않는다.** 예시 설정은 `*.example`로만.

### 발견 (태그 없음)
- 자산은 **frontmatter `description`으로 발견**된다. 등록·태그가 필요 없다 — 발견기가 매번 스캔.
- 대외비 자산은 폴더가 Git에서 빠지므로, 클론에 따라 실제 폴더가 없을 수 있다 → 참조 측은 "없구나" 하고 차선(law.md §6).

### 다른 문서에서 이 자산 참조 (law.md §8)
자연어로 적고 발견기로 찾는다: `관련 자산: OpenAI API 키 — 발견기로 찾아 사용`.

> 자산을 등록·관리하는 자세한 절차는 [[스킬]] `스킬/기본/asset-creator`.
